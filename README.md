# 車両点検リマインダーアプリ「MechaniCare」
## サービス概要

車のメンテナンス時期を忘れがちな車オーナーに
整備士の知識に基づいた適切なタイミングでのリマインド機能を提供する
車両点検管理Webアプリケーション

## このサービスへの思い・作りたい理由

整備士として働いていた経験から、多くの車オーナーが以下のような問題を抱えているのを目の当たりにしてきました：

**実際に見てきた課題**：
- 「オイル交換、いつやったっけ？」→ オイル交換時期超過によるエンジン不調
- 「車検まだ大丈夫だと思ってた」→ 期限切れ直前の慌てた対応
- 「タイヤ交換時期がわからない」→ スリップ事故のリスク
- 「複数台持ってるけど管理が大変」→ メンテナンス漏れ

「もう少し早く来てくれれば、こんなに大変なことにならなかったのに...」と思うことが何度もありました。

**適切なタイミングでのメンテナンスリマインドにより、車の寿命を延ばし、安全な運転をサポートしたい**という思いでこのサービスを作ります。

## ユーザー層について

### **メインターゲット：車を大切にしたい個人オーナー**
**決めた理由**：
整備士として働いていた時に最も多く接したのがこの層。メンテナンス意識はあるものの、「いつ何をすればいいかわからない」という悩みを抱えている人が非常に多かった。

**特徴**：
- 車検は受けるが、日常メンテナンスのタイミングが曖昧
- 「オイル交換っていつだっけ？」「タイヤ交換時期がわからない」が口癖
- メンテナンス費用は惜しまないが、適切なタイミングを知りたい

**ニーズ**：
- 車種に応じた具体的なメンテナンススケジュール
- 忘れないためのタイムリーなリマインド機能
- 整備士の知識に基づいた信頼できる情報

### **サブターゲット：複数台管理が必要な人**
**決めた理由**：
家族の車、仕事用の車など、複数台を管理している人は管理が複雑になりがち。一元管理のニーズが高い。

**特徴**：
- 家族それぞれの車の管理が煩雑
- メンテナンス時期がバラバラで把握しきれない
- 「あの車のオイル交換いつだっけ？」状態

**ニーズ**：
- 複数台の一元管理機能
- 車両ごとの個別通知機能

## サービスの利用イメージ

### **登録・設定フェーズ**
1. 車両情報を登録（車種、年式、走行距離、購入日）
2. 前回のメンテナンス履歴を入力（オイル交換、タイヤ交換等）
3. 通知設定（メール、LINE、アプリ内通知）

### **日常利用フェーズ**
1. **自動リマインド**: 「オイル交換時期です（前回から5,000km経過）」
2. **ダッシュボード確認**: 今月やるべきメンテナンス一覧
3. **実施記録**: メンテナンス完了後、次回予定を自動計算

### **価値提供**
- **予防保全**: 大きな故障を未然に防ぐ
- **コスト削減**: 適切なタイミングでのメンテナンスで長期コスト削減
- **安心感**: 「忘れる心配がない」という精神的な安心
- **知識向上**: 整備士監修のメンテナンス知識を習得

## ユーザーの獲得について

### **整備工場・ディーラーとの連携**
整備士として働いている経験を活かし、整備工場やディーラーに「お客様向けの便利ツール」として紹介してもらう。

**具体的な方法**：
- 同僚や知り合いの整備士に直接紹介
- 「お客さんによく聞かれる『次いつメンテナンスすればいい？』の解決ツール」として提案
- 整備工場の待合室にQRコードを設置

**なぜこの方法を選んだか**：
整備士時代に「メンテナンス時期を忘れるお客さん」を数多く見てきた実体験があるため、現場の整備士に共感してもらいやすく、信頼性のある紹介が期待できる。

## サービスの差別化ポイント・推しポイント

### **整備士の実務経験に基づく精度**
- **一般的な管理アプリ**: 単純な日数・走行距離計算
- **本サービス**: 車種別、使用環境別の最適なタイミング提案

### **多様なリマインド方式**
- **メール通知**: 詳細な内容と推奨整備工場情報
- **LINE通知**: 手軽な確認とワンクリック記録
- **プッシュ通知**: 緊急度に応じた通知頻度調整

### **教育的要素**
- **なぜそのタイミングなのか**: 整備士による解説付き
- **DIYガイド**: 自分でできるメンテナンスの手順
- **トラブル予兆**: 「こんな症状が出たら要注意」情報

### **個別最適化**
- **使用環境考慮**: 都市部/山間部、長距離/短距離使用
- **車種特性**: メーカー・車種別の特有メンテナンス項目
- **ユーザー習慣**: 過去の実施パターンから最適タイミング学習

### **一般的なCRUD以外の実装予定機能**

#### **メール通知システム（メインの差別化機能）**
車両のメンテナンス時期になったら自動でメール通知を送る機能。

**MVPでの実装方針（シンプル版）**：
- **Action Mailer**：基本のメール送信機能（Rails標準）
- **Rakeタスク + cron**：毎日決まった時間にメンテナンス時期をチェック

**MVPでの実装イメージ**：
```ruby
# lib/tasks/maintenance_reminder.rake
namespace :maintenance do
  desc "Send maintenance reminder emails"
  task send_reminders: :environment do
    Vehicle.needs_maintenance_reminder.each do |vehicle|
      MaintenanceMailer.reminder_email(vehicle).deliver_now
    end
  end
end

# app/mailers/maintenance_mailer.rb
class MaintenanceMailer < ApplicationMailer
  def reminder_email(vehicle)
    @vehicle = vehicle
    @user = vehicle.user
    mail(to: @user.email, subject: 'メンテナンス時期のお知らせ')
  end
